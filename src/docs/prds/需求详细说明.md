## 3、合同列表

Contract list 可进行客户，供应商价格合同管理
- 列表字段：**Contract Number**、**Project Name**、**Contract Type**、**Contract Signer、Contract Validity Period、Contract Attachment**、**Contract Status， Fuel Basis，Fuel change frequency**
- 列表排序：
- 默认排序规则：
- 1.按状态正序：**Under Review**，**Active，Pending Effective**，**Expired**，**Rejected**；
- 2. **Under Review**，**Pending Effective**，**Active**按有效期结束时间正序，即最快结束排最前
- 3. **Expired**，**Rejected**按有效期结束时间倒序，即最晚结束排最前
- 列表操作：
- 预览，对合同附件进行预览
- 下载：下载合同附件
- 审核通过，删除
- 操作记录：点击展示操作记录（包含： 创建，审核拒绝（包含拒绝理由description），审核通过）

| 状态 | 操作 |
| Under Review | 预览，下载，审核拒绝，审核通过，操作记录，删除 |
| Pending Effective | 预览，下载，操作记录 |
| Active | 预览，下载，操作记录 |
| Rejected | 预览，下载，操作记录 |
| Expired | 预览，下载，操作记录 |

审核拒绝：需填写拒绝理由；Description限制2000字符，不允许特殊字符，超过不允许继续输入

- 当合同通过审核，contract status= active/Pending Effective， 则创建一个对应类型的price version,；从Pending Effective到active已创建过，则不再创建price version；
- **Customer Price Contract for Logistics Project** 类型则创建 customer price version,
- **Vendor Price Contract for Logistics Project**类型则创建 vendor price version
- 取该合同Contract Validity Period 为对应price version 有效期
- price version 排序规则按照结束期正序，即结束最晚越靠后
- 每创建一个合同，则该合同同步至customer /vendor合同模块进行展示，有状态变更同样同步至customer /vendor/project合同模块，rejected合同不展示至对应customer /vendor
- 搜索：
- 按Contract Number（模糊搜索），project name（模糊搜索）,  Contract Type（单选），Contract Signer（模糊搜索, 当 Contract Type未选择，该搜索框禁用）Contract Status（多选）

### 4.3 价格版本对标准运单的影响

#### 2. 创建运单（标准运单）关于项目有效期的逻辑判断
- 全局隐藏【Quick Dispatch】按钮，业务暂时未使用该功能
- 增加判断in progress状态项目当前时间是否在项目有效期内，不在有效期内则不允许创建运单，【create waybill】时，运单类型字段默认选择【Temporary waybill】，不可修改；【copy waybill】时若有标准运单则copy 失败，提示【Copy failed, only temporary waybills are allowed to be created in non-valid project】
- create waybill：选择项目后，根据项目是否在有效期，限制 Dispatch Type, 若不在有效期内，则Dispatch Type 默认选择【Temporary waybill】，不可修改；若先行选择Dispatch Type ，则选择项目后进行清空；
