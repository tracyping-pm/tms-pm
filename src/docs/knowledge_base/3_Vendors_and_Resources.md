# 知识库：供应商与资源 (Vendors & Resources)

本部分详细说明了 TMS 系统如何管理运输服务所需的核心资源，包括供应商、车辆、司机及其他人力资源。这是确保运输能力和合规性的基础。

## 1. 核心模块

### 1.1. 供应商管理 (Vendor Management)

- **功能**: 对提供运输服务的所有供应商进行全面的生命周期管理。
- **主要操作**:
    - **供应商列表**: 展示所有供应商，并提供搜索功能。
    - **生命周期**: 支持创建、编辑、转移、拉黑/解除、中止/恢复合作等操作。
    - **准入审核 (Accreditation)**: 供应商需要通过资质审核才能被正式启用。
- **详情页切页**:
    - **联系人 (Contacts)**
    - **跟进记录 (Follow-up Records)**
    - **卡车 (Trucks)**: 管理该供应商名下的所有车辆。
    - **司机 (Drivers)**: 管理该供应商名下的所有司机。
    - **助手 (Helpers)**: 管理随车助手等其他人力资源。
    - **项目 (Projects)**: 展示该供应商参与的所有项目。
    - **资质文件 (Accreditation)**: 管理供应商的资质文件。
    - **合同 (Contracts)**: 展示与该供应商签订的所有价格合同。

### 1.2. 车辆管理 (Truck Management)

- **功能**: 独立管理系统中的所有车辆信息。
- **主要操作**:
    - **车辆列表**: 集中展示所有车辆信息。
    - **生命周期**: 支持创建、编辑、停用/启用车辆。
    - **归属管理 (Attribution)**: 将车辆与具体的供应商进行关联。
    - **准入审核 (Accreditation)**: 车辆同样需要上传必要文件并通过审核。

### 1.3. 司机管理 (Driver Management)

- **功能**: 独立管理系统中的所有司机信息。
- **主要操作**:
    - **司机列表**: 集中展示所有司机信息。
    - **生命周期**: 支持创建、编辑、转移、拉黑/恢复司机。
    - **准入审核 (Accreditation)**: 司机需要通过资质审核。

## 2. 关键业务逻辑

- **资源关联关系**: 核心关联是 **供应商 -> 车辆/司机/助手**。一个供应商可以拥有多个车辆和多名司机。运力池（Capacity Pool）通过关联供应商来间接获得运力资源。
- **准入状态 (Accreditation Status)**: 供应商、车辆、司机都有各自的准入状态，直接影响他们是否可以被派单执行运输任务。
