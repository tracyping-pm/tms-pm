# 运单完成逻辑更新

原运单Settled 逻辑：所有结算项均为Collected/Write Off/Paid ，运单财务状态更新为 **settled**

新逻辑：由于业务侧对金额为0的结算项，可能不会关联对账单，故修改逻辑以匹配实际操作习惯
- 若有结算项金额为0，且状态为Verified
- 除以上结算项，剩余所有结算项为Collected/Write Off/Paid 状态；运单财务状态更新为 settled
- 对AR Dashboard 影响：Dashboard计算需剔除状态为Verified且金额为0的结算项对运单统计的影响。（如A运单的A结算项为0，且状态为Verified，其他结算项均为uncollected，则该运单被统计到uncolected, 而不是unbilled）

历史数据处理：
1. 需按照该逻辑进行运单状态刷新,AR dashboard 数据更新
2. 需协助业务处理异常运单：
1. 补充导出格式：[https://docs.google.com/spreadsheets/d/1E1wNjFU32i-QQahIVUP7fpN_TpSiS-XkDhdRQuY4EFg/edit?usp=sharing](https://docs.google.com/spreadsheets/d/1E1wNjFU32i-QQahIVUP7fpN_TpSiS-XkDhdRQuY4EFg/edit?usp=sharing)
2. 导出以下条件交集的运单：
1. 结算项为0，且结算项状态为Verified
2. 运单状态为Awating settlement
3. 该运单其他结算项状态都为Settled（Collected/Write Off/Paid）或 under billing preparation
3. 由业务确认处理方案后，进行统一处理
