# 知识库：通知中心 (Notification Center)

通知中心是 TMS 系统的“神经网络”，负责在关键业务事件发生时，主动向相关人员发送提醒和预警。这确保了信息的及时传递，降低了潜在风险，并提高了团队的协同效率。

## 1. 通知渠道

系统支持两种主要的通知渠道：

1.  **TMS 内部通知**: 在系统右上角的通知中心显示消息，通常带有一个“View Details”按钮，点击后可直接跳转到相关业务对象的详情页面（如项目、合同或运单）。
2.  **Slack 通知**: 将消息推送到指定的 Slack 频道，通常包含核心信息和一个指向详情页的链接，方便用户在移动端或桌面端快速响应。

## 2. 主要通知场景

通知的触发分为系统定时任务和用户操作触发两种。

### 2.1. 预警与到期提醒 (Scheduled Tasks)

系统每天会定时（如 08:00）扫描，检查即将发生或已经发生的事件，并发送通知。

- **项目相关**: 
    - **项目即将过期**: 当项目有效期剩余时间 ≤ 15天时，通知项目相关成员（Pricer, Dispatcher, OC）。
    - **项目已过期**: 在项目过期当天，发送警报，提示无法在该项目下创建标准运单。

- **报价/合同相关**:
    - **客户报价即将过期**: 当客户的最后一个价格版本有效期 ≤ 15天时，通知 Pricer 和客户 BD。
    - **客户报价已过期**: 在过期当天发送通知。
    - **供应商报价即将过期**: 当供应商的某个价格版本有效期 ≤ 15天时，通知 Pricer 和供应商负责人（Vendor PIC）。
    - **供应商报价已过期**: 在过期当天发送警报，提示可能无法计算价格。
    - **油价基准到期**: 根据合同中设置的油价更新频率，在到期时提醒相关人员更新油价。

### 2.2. 关键操作提醒 (Action-Triggered)

在用户执行某些关键操作时，系统会立即发送通知。

- **运单毛利为负**: 
    - **触发条件**: 当用户提交或修改运单，导致系统计算出的毛利 ≤ 0 时。
    - **接收方**: 项目的 Pricer。
    - **内容**: 提示运单亏损，并附上运单链接，要求关注。

- **其他操作**:（在 `功能清单.md` 中提及）
    - 创建/开始/取消/终止项目。
    - 提交/开始/取消/确认运单。
    - 上传 POD、运单异常等。
