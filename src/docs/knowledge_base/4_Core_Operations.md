# 知识库：核心运输运营 (Core Operations)

本部分是 TMS 系统的核心，涵盖了从项目启动、合同签订、价格设定到最终运单执行的全过程。它将系统其他模块（如客户、供应商、财务）串联起来，形成了完整的业务闭环。

## 1. 核心模块

### 1.1. 项目管理 (Project Management)

- **功能**: 项目是业务运营的基本单位，用于组织特定客户的运输任务。
- **生命周期**: `Preparing` -> `In Progress` -> `Terminated`/`Completed`/`Canceled`。
- **核心要素**:
    - **项目有效期**: 定义了项目的起止时间，过期项目无法创建标准运单。
    - **团队成员 (Team Members)**: 为项目指派不同角色（如 Pricer, Dispatcher, OC）。
    - **关联资源**: 项目必须关联 **线路库 (Route Library)** 和 **运力池 (Capacity Pool)** 才能启动。
- **详情页切页**: 聚合了与项目相关的所有信息，如业务单据、合同、线路库、运力池、订单和运单列表。

### 1.2. 合同管理 (Contract Management)

- **功能**: 集中管理与客户和供应商签订的价格合同，是定价和结算的法律依据。
- **合同类型**: `Customer Price Contract` 和 `Vendor Price Contract`。
- **状态流转**: `Under Review` -> `Pending Effective` -> `Active` -> `Expired` / `Rejected`。
- **关键逻辑**: 合同审核通过后，会自动在对应的线路库中生成一个有效的 **价格版本 (Price Version)**。

### 1.3. 线路与定价 (Route & Pricing)

- **功能**: 定义运输的路线和价格，是运费计算的基础。
- **核心概念**:
    - **线路库 (Route Library)**: 包含一个项目下所有的运输路线（起点、终点、途经点）。
    - **价格版本 (Price Version)**: 与合同有效期关联，分离客户价格和供应商价格。一个项目可有多个价格版本。
    - **定价方式**: 支持按路线（Route Billing）和按里程（Mileage Billing）两种方式。
    - **价格维护**: 支持在系统页面直接维护，也支持通过导入/导出 Sheet (Excel) 进行批量管理。

### 1.4. 运单管理 (Waybill Management)

- **功能**: 运输任务的最小执行单位，记录了货物从起点到终点的全过程。
- **运单类型**: **标准运单 (Standard Waybill)** 和 **临时运单 (Temporary Waybill)**。
- **核心操作**:
    - **创建/复制运单**: 可单个创建或批量复制。
    - **车辆调度 (Assign Truck)**: 调度是核心环节，系统会校验路线规划是否完成、车辆状态是否合规、供应商是否有有效的价格版本。
    - **状态跟踪**: 运单有详细的运输状态（如 `Planning`, `Dispatched`, `Delivered` 等）。
    - **POD 管理 (Proof of Delivery)**: 上传和管理签收单。
    - **异常处理**: 记录和处理运输过程中的异常情况。

## 2. 关键业务逻辑

- **价格版本校验服务**: 一个独立的校验服务，用于在创建运单、修改时间、调度车辆等多个环节判断是否存在有效的客户和供应商价格。
- **毛利计算**: 在触发价格计算时（如调度、提交运单），系统会自动计算 **应收 - 应付** 的毛利。如果毛利为负，会发出警告通知。
