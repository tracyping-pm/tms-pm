# 知识库：客户与销售 (Customers & Sales)

本部分主要围绕 TMS 系统中的客户关系管理（CRM）功能，覆盖了从潜在客户（销售线索）到正式客户的全生命周期管理。

## 1. 核心模块

### 1.1. 客户管理 (Customer Management)

- **功能**: 对公司的正式客户进行统一管理和维护。
- **主要操作**:
    - **客户列表**: 提供客户的列表展示和自定义搜索功能。
    - **客户生命周期**: 支持创建客户、编辑客户信息。
    - **所有权转移**: 允许将客户从一个销售或客户经理名下转移给另一人（单个或批量操作）。

### 1.2. 客户详情页 (Customer Details)

- **功能**: 360度全方位展示客户相关的所有信息，是日常客户服务的核心视图。
- **包含切页 (Tabs)**:
    - **基本信息 (Basic Info)**: 客户的主体信息，可编辑和转移。
    - **联系人 (Contacts)**: 管理客户方的联系人信息，支持增删改。
    - **跟进记录 (Follow-up Records)**: 记录与客户的沟通和互动历史，支持增删改。
    - **项目 (Projects)**: 展示该客户下的所有运输项目，并可直接创建新项目。
    - **资质文件 (Accreditation)**: 管理客户的认证或许可文件，如营业执照等，支持上传和下载。
    - **感知/评价 (Perception)**: 记录公司内部对该客户的评价或关键信息，支持增删改。
    - **合同 (Contracts)**: 统一展示与该客户签订的所有价格合同，同步自合同管理模块。

### 1.3. 销售线索池 (Leads Pool)

- **功能**: 管理尚未转化为正式客户的销售机会或潜在客户。
- **主要操作**:
    - **线索列表**: 集中展示所有销售线索，支持搜索和筛选。
    - **线索跟进**: 详情页结构与客户详情类似，允许记录跟进信息。
    - **线索转化**: 支持将线索（Lead）转移（Transfer）给销售人员，当线索成熟后，可以被创建为正式客户（Customer）。

## 2. 关键业务逻辑

- **服务时长计算**: 系统会自动计算客户的服务时长（Service Duration），基于该客户下所有项目的最早开始日期到当前日期。
- **历史合同处理**: 旧系统中分散的客户合同被统一迁移和管理，确保数据一致性。
