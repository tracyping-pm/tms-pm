#### 批量处理工具

更新批量处理工具：原批量处理仅将运单处理至 【In transit】状态，为提升业务效率，故更新工具，将运单处理至【Delivered】状态

Strat Waybill 更新文案至【Confirm Delivered】

处理逻辑：

1. 将导入的运单号：若为planing, pending,In transit ;  则处理为 Delivered 状态
- 同只处理Project 【**SHOPEE-Last mile**】中的运单
- 同样按原逻辑添加shipping record （[https://inteluck.atlassian.net/wiki/spaces/CPT/pages/625803271/Version+3.2+Official+website#1.-%E8%BF%90%E5%8D%95%E8%87%AA%E5%8A%A8%E5%A4%84%E7%90%86%E5%B7%A5%E5%85%B7](https://inteluck.atlassian.net/wiki/spaces/CPT/pages/625803271/Version+3.2+Official+website#1.-%E8%BF%90%E5%8D%95%E8%87%AA%E5%8A%A8%E5%A4%84%E7%90%86%E5%B7%A5%E5%85%B7) ）
1. 若为 其他状态：则不处理，结果行展示 “This status is no longer processed”
2. 需至Drive特定路径 （[https://drive.google.com/drive/folders/1LXbgYmAAYtELJFmRLmBOsRIRpSgItH8t](https://drive.google.com/drive/folders/1LXbgYmAAYtELJFmRLmBOsRIRpSgItH8t) ）根据运单号（ps: 一个运单仅一份POD文件，不考虑多个文件情况）获取该运单 POD 文件，POD存储名称为该Waybill External Code ,将该POD按当前存储规则存放至对应Drive 指定文件夹；若无法获取对应POD，则该行处理失败，提示“Unable to get POD file” ;  按照POD folder 地址寻找该运单对应POD 文件，若文件夹1 未获取到，则顺序到文件夹2进行寻找 （表格样式参考下图）
3. 所获取POD文件需回显至对应Waybill Detail, Type 则取 Others
4. 完成confirm delivery之后，与正常delivered的运单一致，需将对应的运单写入 每日delivered sheet中
