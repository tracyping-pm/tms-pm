# 三，需求说明

## 1. Assign Truck 流程更新

#### 1. 流程变化：先选择vendor，再选择该Vendor下 Truck
- 流程由原来的3步增加为4步，即增加选择供应商逻辑； 每一步的cancel 按钮均更新为save， 即保存当前选项内容，并在waybill detail-- carrier 模块进行展示；关闭该页面及Previous操作不再做二次确认。
- 影响范围：临调+标准运单【临调只是交互调整，数据范围不变，其中状态为Unaccredited 状态不做展示】

- Vendor 数据范围：该项目运力池所有Vendor, 展示该vendor 准入状态及运力池中的access状态
- 排序规则：1. Vendor Access Status 为Approved 优先展示  2. Truck Type Consistency 为一致的优先展示；
- select truck 中，truck的排序同样优先展示可用车辆，1. truck access status 为approved  2. truck status 为Accredited、in use状态优先展示，3. Truck  Type Consistency 为一致的优先展示 5.20 update
- Not Approved的项目置灰， 不可被选中
- vendor展示字段：Vendor Name,    Vendor Status（即该vendor 自身准入状态，5.13 删除该内容，运力池中无该状态vendor），Vendor Access Status（该vendor在运力池中的access status），trucks (vendor下车辆数量），contacts ( 如有多个，仅展示第一个），Truck Type Consistency（该vendor下车辆是否有符合Required Truck Type），允许通过Vendor name 进行like 匹配
- 5.20 增加： 对copy 运单逻辑的影响：
- 若原vendor 当前为 Unaccredited ,not approvaed 状态，则不copy 所有carrier信息
- 若Vendor为 accredited， truck 为 Unaccredited 、 not approved， 则不copy truck 信息，driver 信息 , helper信息；
- 若Vendor为 accredited， truck 为 accredited ，driver 为Unaccredited 、 blocked，则不copy driver 信息，helper信息
- 若helper 被拉黑，仅不copy helper信息

- 点击【save】，保存已选择 信息，退出车辆指派流程，跳转至waybill detail页面，成功Toast 提示【vendor/truck information is saved】,carrier 模块展示已保存信息，且carrier 信息顺序有变化，具体见图 。保存失败则Toast 提示【saved failed】,并停留在当页
- select Driver 不做save 操作，仅允许previous、next 操作

- Select truck 展示逻辑不变，增加字段Access Status（该truck在运力池中的access status）,原不可选择置灰并提示逻辑不变， 删除vendor相关字段及筛选条件；
- 选择driver及 helper 选择逻辑不变，退出逻辑同选择供应商，车辆
- 保存信息后，下次进入Assign truck页面，则跳转至未选择信息页，eg: 已保存vendor 信息，再次进入则跳转至 select truck 页；
- 更新信息则清空该信息之后所保存信息（例如更新vendor， 则已选择并保存的truck，driver，helper信息均进行清空）
- submit 时，需有driver信息（原逻辑保持）, pending及之后状态重新assign 保存时需校验是否有driver信息
